Boy, dictionaries sure are great! They allow us to make a connection between keys and values. We want our smart thermostat AXELA to answer us (in English) how the temperature feels. You assume the coldest it will ever be is -460 ℉ (absolute 0) and the warmest it will ever be is 212 ℉ (boiling point of water). Then you will need to store a connection between temperatures ranges to English words for how you would feel in that range:

-460 ℉ — 40 ℉ 40 ℉ — 60 ℉ 60 ℉ — 80 ℉ 80 ℉ — 212 ℉
Cold Cool Nice Hot

What we want is a soft dictionary, where we specify numeric ranges as the keys (as a tuple), and any value between the left edge (and up to but not including the right edge) should "match" the key. If we asked, "how does it feel?", if it were 45℉, it would say "Cool". If it were 99℉, it would say "Hot". Let's build this!

>>> def SD_create(): return {} ### empty soft dictionary (SD)
>>> def SD_add(SD, low, high, value): SD[(low, high)] = value ### key is tuple
>>> sd = SD_create()
>>> SD_add(sd, -460, 40, "Cold") ### For this entire page, assume the
>>> SD_add(sd, 40, 60, "Cool")   ### left edge value is smaller than the
>>> SD_add(sd, 60, 80, "Nice")   ### right edge value. No need to error check it
>>> SD_add(sd, 80, 212, "Hot")
>>> SD_get(sd, 45)
"Cool"
>>> SD_get(sd, 99)
"Hot"
