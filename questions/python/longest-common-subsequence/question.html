Now we will solve the Longest-Common-Subsequence problem. Given two strings x and y, we define their longest-common-subsequence (or lcs) as the longest substring they have in common, where a substring is defined as a sequence of adjacent characters. For example “ws eat” is a substring of “cows eat yay”, but “coyay” is not.

For example lcs(“crows eat yay”, “my cows eat hay”) is “ows eat ”. As another example, lcs(“pickle”, “orcs”) is “c”. If two strings have no substrings in common, their lcs is the empty string. You may not need all lines. Do not worry about efficiency.

Syntax hints:
Given a bunch of strings, you can use the longest function to return the longest of them, e.g. longest(“let”, “us”, “slather”, “ourselves”, “with”, “pizza”) would return “ourselves”. It also works with a list, e.g. longest([“am”, “ham”, “bell”]) is “bell”.
